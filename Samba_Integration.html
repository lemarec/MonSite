1 - Prérequis :
Avoir un serveur Samba AD (ou Windows serveur) actif et configuré
Avoir un compte administrateur sur ce serveur.
2 - Préparation des clients Windows 10 :
Changer le nom du client Menu démarrer, paramètres. 'Système', en bas à gauche, 'Informations système', sous "spécification de l'appareil', cliquer sur 'Renommer ce PC' Il faut configurer la carte réseau du poste, lui donner une IP sur le sous-réseau du serveur Menu démarrer, paramètres. 'Réseau et Internet', en dessous de 'Paramètres réseau avancés', choisir 'Modifier les options d'adaptateur' Sur la carte réseau affichée, avec le bouton droit sélectionner 'propriétés' Le DNS du client doit être celui du serveur Cliquer sur 'Protocole Internet Version 4' puis plus bas, 'propriétés' Ici, tous les postes clients auront une adresse qui commence par 192.168.2, le serveur précédemment configuré a pour adresse 192.168.2.100, l'indiquer pour la passerelle et le DNS. Cliquer sur 'Valider les paramètres en quittant' puis 'OK'
3 - Intégration au domaine des clients Windows 10 :
Revenir en arrière sur les paramètres jusqu’à 'Comptes', sur le coté choisir 'Accès professionnel ou Scolaire' Cliquer sur '[+] se connecter' Dans la nouvelles fenêtre qui s'ouvre choisir 'Joindre cet appareil a un domaine Active Directory local' Donner le nom du domaine (ici BTSCPI.LAN) puis rentrer login (administrator) et mot de passe de l'administrateur serveur Ensuite donner le login et le rôle d'un utilisateur (utilisateur standard ou administrateur) qui va utiliser le client Il faut ensuite redémarrer. l'ordinateur sera sur le domaine. Sur Windows 7 la démarche est sensiblement identique, il faut aller dans 'panneau de configuration', 'Système et sécurité', 'Système', a droite du nom de l'ordinateur, cliquer sur 'Modifier les paramètres', cliquer sur 'Modifier' pour renommer ou changer de domaine, membre d'un -> Domaine, remplir le nom de domaine puis 'OK'
4 - Gestion de l'horloge - Temps internet :
Il faut assurer une bonne synchronisation des horloge des clients avec le serveur Linux. Aller dans le menu redémarrer et taper "cmd", a droite cliquer sur "exécuter en tant d'administrateur" Exécuter les commandes suivantes : w32tm /config /syncfromflags:DOMHIER /update w32tm /resysnc /nowait net stop w32time net start w32time si cela ne marche pas ou s'il y a des erreurs on peut vérifier la configuration par : w32tm /query /source w32tm /monitor On doit parfois utiliser la commande "net time" : net time /set Répondre oui pour forcer la synchronisation.
