1 - DONNEES DU SERVEUR (à choisir au départ) :

Distribution Linux utilisée ici :Ubuntu 20.0411/2020
Nom de la machine :ServeurLinux15 octets maxi, Attention pour caractères UTF-8 qui dépassent un octet.
Adresse IP du serveur :192.168.1.200
Adresse IP de la passerelle (qui fournit Internet) :192.168.1.1
Terminaison du suffixe DNS : .lanIci pas de DNS d'entreprise ou de domaine acheté, .local réservé par Apple !!!!!
Domaine de recherche : btscpi.lan
REALM :BTSCPI.LAN
FQDN :ServeurLinux.btscpi.lanFully Qualified Domain Name, max 63 octets / étiquette, max 255 octets / nom domaine complet

2 - CONFIGURER LE NOM DNS :

a/° Modifier le fichier /etc/hostname et mettre le FQDN du serveur

# srvads.mydomain.lan
LinuxServer.btscpi.lan

b/° Modifier le fichier /etc/hosts et y mettre le FQDN et nom court

# ne pas modifier la ou les lignes concernant localhost
192.168.1.200 LinuxServer.btscpi.lan LinuxServer

3 - CONFIGURER L'ADRESSE IP :

Pour configurer un serveur samba il faut une adresse IP fixe.

ip addr
ou
ifconfig -a

permettent d'obtenir le nom des interfaces et l'état de celles-ci.
Pour configurer le réseau il existe plusieurs gestionnaires de connexion sous linux ( NetworkManager(Ubuntu),NetPlan (Ubuntu 18.04 +), ifup (Debian), ....) Configuration classique : les fichiers sont dans /etc/network/interfaces Pour éviter les conflits on désactive les outils graphiques : apt remove network-manager wicd #debian apt-get remove network-manager wicd #Ubuntu Exemple de fichier /etc/network/interfaces # /etc/network/interfaces auto eth0 iface eth0 inet static address 192.168.0.200/24 gateway 192.168.0.1 NetworkManager : On utilise l'interface graphique NetPlan : les fichiers sont dans /etc/netplan/*.yaml Exemple de fichier de config netPlan, cat /etc/netplan/*.yaml # Let NetworkManager manage all devices on this system network: version: 2 renderer: NetworkManager Finalement ici c'est NetworkManager qui gère !!!!! Attention si NetworkManager tourne, le fichier /etc/resolv.conf change sans arrêt ce qui va rendre impossible le fonctionnement du DNS de Samba. On passe à une gestion par networkd en modifiant l'option 'renderer' : # This file describes the network interfaces available on your system # For more information, see netplan(5). network: version: 2 renderer: networkd ethernets: enp0s31f6: dhcp4: yes Ici dhcp sur l'interface enp0s31f6 network: ethernets: eth0: addresses: [192.168.1.200/24] gateway4: 192.168.1.1 nameservers: addresses: [8.8.8.8, 8.8.4.4] dhcp4: false dhcp6: false version: 2 Application des modifications :
sudo netplan apply
systemctl restart systemd-networkd

ou

reboot

4 - AUTRES CONFIGURATIONS :

Langue du système : En anglais pour faciliter la recherche de problèmes dans les logs

apt-get install -y locales-all
localectl set-locale LANG=en_US.utf8
localectl status

Désactiver le demon Avahi (peut perturber la resolution des noms

systemctl stop avahi-daemon.service avahi-daemon.socket
systemctl disable avahi-daemon.service avahi-daemon.socket

Mettre a jour et installer des paquets indispensables

apt-get update -y
apt-get install -y wget sudo screen nmap telnet tcpdump rsync net-tools dnsutils htop apt-transport-https vim gnupg
